# Architecture Decision Records

Dieser Bereich sammelt dauerhafte Architektur- und Designentscheidungen.

ADRs sind für Entscheidungen gedacht, die verändern, wie das System strukturiert, integriert oder betrieben wird.

Jedes ADR sollte enthalten:

- Titel
- Datum
- Status
- Kontext
- Entscheidung
- Konsequenzen
