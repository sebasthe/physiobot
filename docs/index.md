# PhysioBot Documentation

This site is the stable reference for how PhysioBot is structured and why key design choices were made.

It is intentionally limited to durable technical documentation:

- architecture overviews
- service and integration maps
- runtime and data-flow diagrams
- architecture decision records

It intentionally excludes planning material such as implementation plans, backlog items, sprint notes, and scratch documents.

## Start Here

- Read the [Architecture](architecture/index.md) section for the system overview and service map.
- Read the [Decisions](adr/index.md) section for architecture and design decisions.

## Scope

The primary audience is product and technical stakeholders who need to understand the system without reading the codebase in depth.
